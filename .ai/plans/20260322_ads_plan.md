# Plan: AI Promotional Ad Generation (Gemini)

## Overview & Feasibility
**Feasibility: Very High (⭐⭐⭐⭐⭐)**

Using Google Gemini to generate high-quality promotional material (ads) tailored to specific platforms (Facebook, Instagram, WhatsApp) is an excellent use case for generative AI. Since Cabox already has the `GEMINI_API_KEY` configured for product ingestion (from photos), adding a feature to synthesize marketing copy based on the product's attributes is highly feasible and requires minimal setup.

## User Experience (Admin Only)

**Location:** 
At the bottom of the Product Edit page (`/admin/products/[id]/edit`), directly after the "SEO e identificadores" section.

**Interaction Flow:**
1. The administrator navigates to the bottom of the product page where they see a **"Material Promocional (AI)"** card.
2. The UI presents radio buttons or a toggle group for the target platform:
   - 🔵 **Facebook** (Focuses on engagement, longer narrative, link previews)
   - 🟣 **Instagram** (Focuses on aesthetics, emojis, and hashtags)
   - 🟢 **WhatsApp** (Focuses on direct communication, concise details, actionable pricing/links)
3. The admin clicks the **"✨ Generar Anuncio con IA"** button.
4. An API route `/api/ai/generate-ad` receives the product payload (name, description, price, features) and the selected platform.
5. Gemini 2.0 returns a highly optimized markdown/text-based ad.
6. The generated ad is displayed in an editable `textarea`, allowing the admin to further adjust the tone before hitting "Guardar".

## Data Architecture

Since these ads should "remain associated to the product... but only visible to the administrator", the best approach is to store this data without cluttering the main storefront APIs. 

**Database Changes:**
We have two viable options to store these generated ads.
* **Option A (Adding JSON field to Product):** Add an `adCampaigns` field of type `Json` to the `Product` model. It could store `{ facebook: "...", instagram: "...", whatsapp: "..." }`.
* **Option B (Dedicated Table):** Add a `ProductAd` table related to `Product`. Best if there might be multiple ad variations per platform in the future.

*Recommendation:* **Option A** is simpler, very performant, and perfectly meets the current need.

```prisma
model Product {
  // ... existing fields ...
  
  // Promotional copy generated by AI (visible only to admin)
  promotionalCopy Json? // e.g. { "instagram": "texto...", "whatsapp": "texto...", "facebook": "texto..." }
}
```

## System Architecture

1. **Frontend Component (`AdGenerator.tsx`)**
   - Resides in `app/src/components/admin/AdGenerator.tsx`.
   - Fetches generation from the backend API.
   - Contains a quick "Copy to Clipboard" button so the admin can instantly drop it into WhatsApp Desktop or Meta Business Suite.

2. **Backend API (`/api/ai/generate-ad/route.ts`)**
   - Validates the admin session.
   - Constructs a strict _System Prompt_ based on the platform.
   - E.g. **System Prompt for WhatsApp:** *"You are an expert Costa Rican digital marketer. Generate a WhatsApp sales message for this product. Use a direct, friendly tone, plenty of emojis, mention the price (${price} ₡), and add a clear call to action to buy. Keep it under 150 words."*
   - Sends the prompt to `gemini-2.5-flash` or `gemini-2.0-pro-exp` depending on the desired tone.
